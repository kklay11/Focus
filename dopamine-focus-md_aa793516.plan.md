## 多巴胺脱敏专注产品 Markdown 文档规划

### 目标

为你当前的产品想法撰写一份结构化、可直接落地的中文 Markdown 文档（如 `focus-detox-spec.md`），用于：

- **清晰描述问题与原理**：多巴胺脱敏、脑雾、考试专注困难的机制说明。
- **整理产品方案**：将“低刺激窗口”、“枯燥练习”、“考试微刺激替代”设计成产品功能模块。
- **给后续开发用作蓝本**：前期可以只是文档，后续可据此实现网页或 App。

### 文档结构规划

文档将采用如下大纲（可直接作为 Markdown 目录）：

1. **产品概述（Problem & Vision）**  

- 现代人的多巴胺脱敏问题：短视频、刷手机、信息过载  
- 典型表现：注意力分散、脑雾、对枯燥任务极度抗拒  
- 产品愿景：通过行为训练与环境控制，重建大脑的“低刺激耐受度”和深度专注能力

2. **核心神经科学与心理学原理**  

- 多巴胺基准线：早晨低位、频繁高刺激如何拉高基准线  
- 前额叶皮层与冲动控制：为什么需要“枯燥练习”  
- 考试场景下的脑雾与“微刺激”重置机制（痛觉→去甲肾上腺素→短暂清明）

3. **目标用户与使用场景**  

- 目标用户画像：
 - 备考学生（中考、高考、考研、各类资格考试）
 - 长时间使用电子设备的知识工作者  
- 使用场景：
 - 日常作息管理（起床与睡前）
 - 日常专注训练（学习/工作日）
 - 考试 / 模拟考试场景

4. **功能模块一：低刺激窗口（Morning & Night Low-Stim Window）**  

- 规则描述：
 - 起床后 1 小时、睡前 1 小时禁止使用电子屏幕  
- 原理说明（简洁科普）  
- 产品实现思路：
 - 网页版：
 - 使用提醒式、任务式引导（无法彻底“锁机”，但可做行为约束与记录）
 - App 版（可选扩展）：
 - 自定义低刺激时间段  
 - 倒计时界面 + 行为记录（打卡、填写替代动作）
- 替代动作引导：
 - 拉伸/简单体操  
 - 冷水洗脸或冷水刺激  
 - 记录当天 3 个核心任务 
- 数据记录与反馈：
 - 每日是否完成低刺激窗口  
 - 早晚各自完成度与连胜天数

5. **功能模块二：枯燥练习（Boredom Training）**  

- 训练目标：提高大脑对“无刺激状态”的耐受、强化前额叶的控制力  
- 练习形式示例：
 - 盯着墙壁 / 一个固定点 5–10 分钟  
 - 数呼吸（专注呼吸感受）  
 - 手写抄录无意义文字/文段  
- 产品实现思路：
 - 训练配置：
 - 每日训练时长（5–10 分钟）  
 - 训练类型选择  
 - 专注计时界面：
 - 大倒计时 + 简单视觉元素（尽量低刺激）  
 - 训练中禁止切出/中断时给出温和提醒  
 - 完成反馈：
 - 记录总训练时长、连续天数  
 - 简单主观打分：训练前/后专注感  
- 游戏化建议：
 - 完成一定次数解锁“耐受徽章”  
 - 累计训练时长可视化（如“你已经忍受了 X 小时的无聊”）

6. **功能模块三：考试中的微刺激替代（On-Exam Micro-Reset）**  

- 问题场景：
 - 考试中脑雾、思维停顿、想摸手机但不可能  
- 原理总结：
 - 轻微痛觉 → 多巴胺 + 去甲肾上腺素短暂提升 → 意识清明 5–10 分钟  
- 推荐动作：
 - 用力按压虎口 5 秒  
 - 用指甲掐指甲床 5 秒  
- 产品实现思路（主要是“课前/考前教育 + 复盘记录”）：
 - 考前预演模块：
 - 教程式页面：图示 + 文本讲解动作与注意事项（避免过度用力伤害）  
 - 模拟练习：在模拟答题训练中，加入“脑雾”自我标记和“微刺激重置”步骤  
 - 考后/练习后复盘：
 - 本次考试/模拟中使用了几次“微刺激重置”  
 - 使用后是否感到清明（主观 1–5 分评分）

7. **应用整体使用流程设计**  

- 新用户第一次使用路径：

 1. 简短测评：当前专注问题、自控力、使用电子设备习惯  
 2. 自动生成 7 天或 14 天“多巴胺重启”计划  
 3. 引导设置：早晚低刺激窗口 + 每日枯燥练习  

- 每日例行流程：
 - 早上打开应用 → 查看今日 3 个核心任务 → 完成低刺激窗口替代动作并打卡  
 - 白天/晚上 → 启动一次枯燥练习 5–10 分钟  
 - 考试周或模拟考试 → 使用“微刺激预演 + 复盘”模块

8. **激励与反馈机制**  

- 数据看板：
 - 连续低刺激天数、累计枯燥练习时长、考试/模拟中清明次数  
- 成就系统：
 - 如“连续 7 天早起不刷手机”、“累计 60 分钟枯燥练习”等  
- 主观反馈：
 - 每周一次：自评专注度、脑雾频率、使用手机冲动程度

9. **隐私与边界说明（简要）**  

- 不采集敏感个人内容（如考试成绩、具体试卷内容），只记录行为数据与主观评分  
- 在文档中写明：本产品为行为训练和自我管理工具，不替代任何医疗诊断与治疗

10. **后续扩展与技术实现建议（可选）**  

 - 网页版：
 - 可用技术栈建议（如 React/Vue + 简单后端 API）  
 - App 版：
 - 可考虑采用跨平台方案（如 React Native / Flutter）  
 - 与番茄钟/任务管理工具的整合设想
